# 폰 노이만 병목 현상과 시스톨릭 어레이 아키텍처 효율성 비교 실험 코드

본 프로젝트는 **아두이노 나노** 5대를 이용해 단일 보드(폰 노이만)와 2×2 격자
시스톨릭 어레이(WS/OS)의 처리 시간을 측정·비교하기 위한 실험 코드입니다.

## 🔌 배선 요약 (2×2 격자)

```
       (0,0) ── A ──▶ (0,1)
         │              │
         B              B  (OS)   /   P  (WS)
         │              │
         ▼              ▼
       (1,0) ── A ──▶ (1,1)
```

- **CLOCK_A / DATA_A** : D2, D4 → A 데이터용 (수평 방향)
- **CLOCK_B(또는 P) / DATA_B(또는 P)** : D3, D5 → 세로 방향
- **D6 (TIMING_TRIG)** : PE(0,0)의 D6를 PE(1,1)의 D6에 직결 → 시작 시각 트리거
- **GND 공통 접지** 필수
- **I2C LCD** : A4(SDA), A5(SCL)

## ⚙️ 실험 방법

1. `01_SingleBoard_VonNeumann.ino` 를 단일 보드에 업로드 → LCD로 처리시간(µs) 확인
2. `02_OS_Systolic` 의 4개 파일을 각각 해당 위치의 보드에 업로드
   - **먼저** PE01, PE10, PE11 을 리셋 상태로 두고
   - **마지막에** PE00 를 리셋 → 3초 대기 후 자동 시작
   - PE(1,1) LCD에 `C11=..., T(us):...` 로 결과 표시
3. `03_WS_Systolic` 도 동일한 순서로 실험
4. 행렬 크기 변경 시 각 `.ino` 상단의 `K_SIZE` 및 `A_stream / B_col*` 배열을 수정

## 📊 측정 지표

- `micros()` 로 측정한 처리 시간 (마이크로초)
- 단일 보드는 메모리 접근 횟수도 함께 출력 (Serial Monitor)
- 이론값과의 오차 계산에는 `Nc = M + N + K - 2` (2×2 격자 기준) 공식 사용

## ⚠️ 주의사항

- 모든 보드의 **GND는 반드시 공통**으로 묶어주세요.
- 아두이노 나노 CH340 호환보드 사용을 가정합니다.
- LCD가 없거나 다른 주소(0x3F)인 경우 상단 `#define LCD_ADDR` 값을 수정하세요.
- 클럭 지연(`CLK_HALF_US = 20`)이 너무 짧으면 통신이 불안정할 수 있습니다.
  안정성이 우선이면 30~50 으로 늘려주세요.

## 📌 데이터 값 (2×2×2 기준)

- 행렬 A = `{{1,2},{5,6}}`
- 행렬 B = `{{1,2},{5,6}}`
- 이론 결과 C = `{{11,14},{35,46}}`
